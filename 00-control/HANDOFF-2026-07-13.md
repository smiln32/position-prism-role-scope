## What this session shipped (all merged to master 2026-07-13)

Additive throughout; frozen `schema.ts` untouched; integrity rules intact.

- **PR #4 — data-at-rest encryption.** Passphrase-protected localStorage behind
  the `StorageLike` seam. `app/src/project/crypto.ts` (WebCrypto
  PBKDF2(250k)/AES-GCM, no new deps), `app/src/project/vault.ts`
  (`EncryptedStorage`: in-memory decrypted working copy, serialized
  encrypt-through, `enable`/`unlock`/`disable`/`flush`/`exportSealed`),
  `App.tsx` unlock gate + home-screen "Protect this computer" panel. Passphrase
  and key are memory-only (rule 4). User doc: `08-docs/SECURITY.md`.
- **PR #5 — list-field editing.** Array fields (a process's steps, a decision's
  criteria, thresholds, "who else knows", a system's quirks) editable
  item-by-item via `addListItem`/`editListItem`/`removeListItem`/
  `listFieldValues` in `capture.ts`, with an inline editor in
  `KnowledgeScreen.tsx`. Steps renumber 1..n; `removeListItem` is the only
  removal — owner-directed, attributable (bumps `updatedAt`), never a silent
  drop.
- **PR #6 — storage durability.** Collision-proof `newId` (full 122-bit UUID,
  was 32 bits) + a one-deep backup slot in `store.ts` (`BACKUP_PREFIX`): save
  backs up the current *valid* primary; load recovers from backup on a
  missing/corrupt primary; quota failure → clear error with the prior primary
  intact; remove clears the backup. `mergeModels()` intentionally still NOT
  wired (deferred until a real sync path — wiring it now would be build-ahead).
- **Merge reconciliation (#4 × #6).** The vault now also encrypts #6's
  `successor:project-backup:` keys (`isManagedKey` = project OR backup prefix),
  so backups are sealed at rest and recoverable. See DECISIONS.md 2026-07-13
  "Gate / Integration".
