- **PR #9 — scroll-to-top on navigation** (UX fix). The app is a single-page
  screen switch, so the browser kept the prior scroll position across every view
  swap and a new (often shorter) view opened part-way down — reported when
  "Start a new project" opened mid-page. One class of bug, three places, fixed
  the same way (`window.scrollTo(0,0)` in a `useEffect` keyed on the view id):
  `App.tsx` (keyed on `screen`, covers all top-level nav), `DeliverablesScreen`
  (keyed on `openId`, list ↔ read swap), `InterviewScreen` (keyed on `trackId`,
  picker ↔ questions swap). Deliberately NOT applied where a scroll would harm
  (answering a question keeps the reader on the feedback line; the Knowledge
  "+ Add" form expands in place). See DECISIONS.md 2026-07-14.
- **PR #10 — top-level README + `.env` gitignore hardening** (docs/hygiene).
  New `README.md` (what Successor is, how it works, how a new person runs it),
